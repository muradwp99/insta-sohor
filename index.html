1) How does javascript works?
Answer: JavaScript is a programming language additionally called the client-aspect scripting language. That means it usually performs tasks on user machines in web browsers. Whenever any programmer writes any code the source code has to pass through so many steps to run. For a fact, javascript doesn't have to pass through any compiler to translate source code it has an interpreter in the browser to reads each line of the JS code and execute it inside the browser. In modern browser it doesn't run Javascript code directly, nowadays every Browser has its own JavaScript engine to run js code. For example, Google Chrome has a V8 engine to perform code.


2) Difference Between Local Storage and Session Storage

Answer:

Local Storage Session Storage
1. Data stored in the Local Storage has no expiration time or date. It stores data after close, reloading, or revisiting the browser or tab. 1. Session Storage stores data only for sessions which means data will be removed after closing or refreshing the browser.
2. Local Storage's data remain on a particula origin that means a particular protocol or host: port will be provided 2. Session storage's data is specific for the protocol of the page for example when accessing any HTTP or HTTPS request for sam web page facebook.com has different session storage.
3. In Local Storage when we store data it isn't sent to the server that's why you can store more data in Local Storage. 3. When we store any data in session storage it requests to the server that it's only accessible when this page is requested from a server.
